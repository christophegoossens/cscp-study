Seven Basic Tools of Quality

The seven basic tools are graphical/visual tools for understanding and improving processes. They require no advanced statistics and can be used by front-line workers.

1. Process Map (Flowchart)

Process map (flowchart): a diagram of the flow of a production or service process through the production system. Standardized symbols represent operations, decisions, flows, and delays. The starting point for most process improvement efforts.
Exhibit 4-38: Process Map of Order Fulfillment Process

2. Control Chart

Control chart: a graphic comparison of process performance data with computed control limits. The process data are displayed as a time series of points. Differentiates between common cause variation (random, inherent to process) and special cause variation (assignable, requires investigation).
  • UCL (Upper Control Limit) and LCL (Lower Control Limit) are set statistically (typically ±3 sigma from the mean)
  • Points outside the control limits or non-random patterns indicate special causes
  • Distinguish from specification limits: control limits are process-based; spec limits are customer-defined
Exhibit 4-39: A Control Chart

3. Pareto Chart

Pareto chart: a bar graph that displays the results of a Pareto analysis (80/20 rule). Bars are arranged in descending order of frequency or impact. Identifies the "vital few" causes that account for the majority of defects or problems.
Exhibit 4-40: Pareto Chart

4. Cause-and-Effect Diagram (Fishbone / Ishikawa)

Links an effect with its possible causes. Common categories (5Ms): Machines, Methods, Materials, Measurements, Manpower (People). Also includes Environment. Used to identify root causes using the "Five Whys" technique.

Exhibit 4-41: Cause-and-Effect Diagram

5. Histogram

A bar graph showing the frequency distribution of measurement data. Reveals: natural vs. unnatural distributions, whether variation falls within specification limits, and whether data is skewed. Unlike the Pareto chart, a histogram shows frequency distribution of measured values, not frequency ranking of categories.

Exhibit 4-42: Histogram Example

6. Check Sheet

A simple tally-count tool for recording event occurrences (defect types, error frequencies). The starting point for CI projects — provides raw data for other tools. Easy to use at the point of production without special skills or software.

Exhibit 4-43: Check Sheet

7. Scatter Diagram

Shows the relationship between two variables. A diagonal pattern of dots indicates correlation (positive or negative). Does NOT prove causation — only correlation. Useful for testing hypotheses from cause-and-effect analysis.

Exhibit 4-44: Scatter Diagram (Positive Correlation)
Seven New (Management) Tools of Quality

Developed in the mid-1970s by Japanese engineers to handle non-numerical, language-based planning and quality data. More useful for planning and design phases.

1. Affinity Diagram

Organizes a large number of brainstormed ideas into natural groupings. Each group is given a descriptive title. Used for complex problems with many unstructured ideas; helps teams discover patterns.

Exhibit 4-45: Affinity Diagram

2. Tree Diagram

Lists tasks in increasingly finer detail to meet a specific goal. Shows hierarchical breakdown from goal to sub-tasks to specific actions. Similar to a WBS (work breakdown structure).

Exhibit 4-46: Tree Diagram

3. Matrix Diagram

Shows relationships between two or more groups of information, the strength of the relationship, and the role of each group. Used to identify which causes most strongly affect which effects, or to map customer requirements to product features.

4. Process Decision Program Chart (PDPC)

Visually captures things that might go wrong in a plan and identifies countermeasures. Used in two ways: (1) to identify preventive measures for potential failures; (2) to brainstorm contingency responses.

Exhibit 4-48: Process Decision Program Chart

5. Relationship Diagram (Interrelationship Diagram)

Illustrates cause-and-effect relationships among factors. Uses arrows to show which factors cause others. Goal: identify the major cause (most arrows out) and the most interrelated factor (most arrows in + out).

Exhibit 4-49: Relationship Diagram

6. Matrix Data Analysis Chart

Shows relationships between groups of information using quantitative data plotted in a matrix format. More analytical than the matrix diagram — uses actual data points to reveal clusters and patterns.

Exhibit 4-50: Matrix Data Analysis Chart

7. Activity Network Diagram (Arrow Diagram)

Shows the sequence of tasks and dependencies in a project plan. Also called a critical path method (CPM) chart. Identifies which tasks can be done in parallel and which must be sequential; finds the critical path.

Exhibit 4-51: Activity Network Diagram
Choosing the Right Tool
  • Nonquantitative tools: process flowchart, cause-and-effect diagram, affinity diagram, tree diagram, PDPC, relationship diagram
  • Quantitative tools: scatter diagram, Pareto chart, histogram, control chart
  • Combination tools: matrix diagram (can be either)
If one tool doesn’t yield actionable information, try another. The tools are complementary — e.g., use a check sheet to gather data, a Pareto chart to identify the top problem, a cause-and-effect diagram to find root causes, and a control chart to verify the fix worked.

What is a Control Chart used for?
ANSWER
A graphic comparison of process data vs. computed control limits over time. Differentiates special cause variation (assignable, investigate) from common cause variation (random, inherent to process). Points outside UCL/LCL indicate special causes.
TERM / CONCEPT
What is the difference between control limits and specification limits?
ANSWER
Control limits are statistically derived from the process itself (±3 sigma); they indicate process stability. Specification limits are set by the customer and define acceptable product. A process can be in control but still produce out-of-spec parts.
TERM / CONCEPT
What is a Pareto Chart?
ANSWER
A bar graph showing defect or problem categories in descending order of frequency or impact. Based on Pareto’s 80/20 principle — identifies the 'vital few' causes that account for most problems so resources focus where they have the most impact.
TERM / CONCEPT
What is a Cause-and-Effect Diagram?
ANSWER
Also called a fishbone or Ishikawa diagram. Links an effect (problem) with its possible causes, organized by categories (5Ms: Machines, Methods, Materials, Measurements, Manpower). Used with Five Whys to find root causes.
TERM / CONCEPT
What is the Five Whys technique?
ANSWER
A root cause analysis method that involves asking 'Why?' repeatedly (typically 5 times) for each answer, until the root cause is revealed rather than a symptom. Used with the cause-and-effect diagram.
TERM / CONCEPT
What is the difference between a Histogram and a Pareto Chart?
ANSWER
Histogram: shows frequency distribution of measured values (bell curve, skew, outliers). Pareto: ranks categories of defects/problems by frequency — uses discrete categories, not a continuous distribution. Both are bar charts but answer different questions.

How are the 14 quality tools categorized?
ANSWER
Nonquantitative: process flowchart, cause-and-effect, affinity, tree, PDPC, relationship diagram. Quantitative: scatter, Pareto, histogram, control chart. Combination: matrix diagram (can be either depending on use).
TERM / CONCEPT
What is common cause variation vs. special cause variation?
ANSWER
Common cause: random variation inherent to the process — always present, managed by changing the process. Special cause: assignable variation from specific, identifiable sources — investigated and eliminated. Control charts distinguish the two.
