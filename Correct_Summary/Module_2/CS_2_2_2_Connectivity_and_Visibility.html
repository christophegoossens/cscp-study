Information System Architecture
Information System Architecture: a model of how the organization operates regarding information. Considers four factors: (1) information flow, (2) data and hardware, (3) applications, and (4) operations management and IT staff.
  • Database: structured repository of data. Managed by a DBMS (Database Management System).
  • Data dictionary: catalog of requirements/specifications for an IS, or a file that stores facts about data itself (metadata).
  • Network: interconnection of computers, terminals, and channels to facilitate file and peripheral sharing. LAN = local; WAN = wide area; VPN = secure internet tunnel.
  • Intranet: private network using internet technology, for internal use. Extranet: network connection to a partner’s network using secure internet protocols.

SaaS & Cloud Computing
Software as a Service (SaaS): a software licensing and distribution model that provides access to applications via the internet on a subscription basis.
  • No local installation or download. Logic and data stored centrally; users access via browser.
  • Benefits: eliminates installation/upgrade time, lower upfront cost, always on latest version.
  • Cloud computing: IT resources (storage, compute, software) delivered over the internet. Mainstream since 2020. Supports SaaS, PaaS, and IaaS models.
  • SC software benefits from cloud: track-and-trace, supplier quality metrics, WMS inventory aggregation, API-based integration, automated regulatory forms.

E-Commerce & VMI
  • B2B e-commerce: electronic transactions between businesses. Includes EDI, electronic marketplaces, and web-based procurement portals.
  • B2C e-commerce: direct-to-consumer transactions via websites or apps.
  • VMI (Vendor Managed Inventory): the supplier monitors the customer’s inventory levels and replenishes automatically. Reduces stockouts and ordering workload for the customer.
  • Consignment: supplier owns inventory at the customer’s location until it is used/sold. Reduces customer working capital tied up in inventory.

ASCM def: Network
ANSWER

TERM / CONCEPT
What is a LAN?
ANSWER
Local Area Network: a high-speed data communication system for linking computer terminals, printers, and databases in close proximity. Used within a building or campus.
TERM / CONCEPT
What is a WAN?
ANSWER
Wide Area Network: connects computers across geographically dispersed locations (multiple buildings, cities, countries). Used to share information between facilities.
TERM / CONCEPT
What is a VPN?
ANSWER
Virtual Private Network: a low-cost internet-based secure transmission method that allows secure data exchange over the public internet — acts like a private network.
TERM / CONCEPT
What is the difference between intranet and extranet?
ANSWER
Intranet: privately owned network using internet technology for internal use only (employees). Extranet: a network connection to a partner’s network using secure internet protocols — extends internal systems to select external parties.
TERM / CONCEPT
What is a DBMS?
ANSWER
Database Management System: software designed for organizing data and providing the mechanism for storing, maintaining, and retrieving data. Examples: Oracle, SQL Server, MySQL.
TERM / CONCEPT
What is a data dictionary?
ANSWER
Either (1) a catalog of requirements and specifications for an information system, or (2) a file that stores facts about data itself (metadata) — field names, data types, relationships.
TERM / CONCEPT
What are 3 advantages of SaaS?
ANSWER
1. Eliminates installation and upgrade time (always on latest version). 2. Lower upfront cost (subscription vs. license). 3. Accessible from anywhere via browser — no hardware to maintain.
TERM / CONCEPT
What SC benefits does cloud computing enable?
ANSWER
Real-time track-and-trace across partners, supplier quality metric tracking, automated regulatory forms, API-based integration, WMS inventory aggregation, and faster scalability of SC systems.
TERM / CONCEPT
What is VMI (Vendor Managed Inventory)?
ANSWER
A system where the supplier monitors the customer’s inventory levels and replenishes automatically when stock falls below a threshold. Reduces stockouts and customer ordering workload.
TERM / CONCEPT
What is consignment inventory?
ANSWER
The supplier owns the inventory while it is stored at the customer’s location. The customer pays only when they use or sell it. Reduces the customer’s working capital tied up in inventory.
TERM / CONCEPT
What is B2B e-commerce?
ANSWER
Electronic commerce between businesses: EDI transactions, electronic marketplaces, procurement portals, supplier catalogs. Automates purchasing, reduces transaction costs.
TERM / CONCEPT
What is B2C e-commerce?
ANSWER
Electronic commerce directly with consumers via websites or apps. Requires last-mile logistics, digital payment systems, and often individual-order fulfillment capabilities.
TERM / CONCEPT
What is client/server configuration?
ANSWER
The most common IT configuration — clients (PCs/devices) send requests to servers (which process and store data). The server provides a specific service to multiple client devices.
TERM / CONCEPT
What is the operating system (O/S)?
ANSWER
A set of software programs that control the execution of hardware and application programs. Manages resources (CPU, memory, storage). Examples: Windows, Linux, Unix, MacOS.
TERM / CONCEPT
What is application software?
ANSWER
Software controlled by an OS that fills specific computing needs (word processing, ERP, WMS, etc.). Runs on top of the operating system.
TERM / CONCEPT
What is the difference between SaaS and conventional software?
ANSWER
Conventional: user purchases license, installs on own hardware, manages upgrades. SaaS: vendor hosts and manages software; user pays subscription and accesses via internet — no local installation.
TERM / CONCEPT
How does connectivity between SC partners work technically?
ANSWER
Via networks (LAN/WAN/VPN), APIs, EDI, or web portals. An extranet or cloud platform allows multiple partners to exchange data securely. APIs are becoming the dominant integration method.

What is the benefit of end-to-end supply chain visibility?
A.It eliminates all supply chain disruptions
B.It allows all parties to see real-time status of inventory, orders, and shipments across the network — enabling faster exception detection and response

End-to-end visibility gives supply chain managers a complete picture of what is happening across the entire network — enabling faster identification of problems and more effective response before they impact customers.
Question 2
What is the Internet of Things (IoT) and how does it contribute to supply chain visibility?
A.A network of retail websites connected for e-commerce
B.A network of physical devices (sensors, RFID tags, GPS trackers) connected to the internet that continuously transmit data about their status, location, and condition — providing real-time supply chain visibility
C.A type of social media platform for supply chain managers
D.A cloud-based ERP system
IoT devices (temperature sensors in refrigerated trucks, GPS on containers, RFID tags on pallets) transmit continuous real-time data, providing unprecedented visibility into the condition and location of goods throughout the supply chain.
Question 3
What does 'supply chain connectivity' mean?
A.Physical connections between distribution centers via roads
B.The ability of supply chain systems and partners to exchange data seamlessly — through EDI, APIs, portals, or cloud platforms — enabling coordinated decision-making
C.The number of countries in which a company operates
D.The bandwidth of a company's internet connection
Connectivity refers to the technical and organizational ability to share data across supply chain partners and systems — the foundation of supply chain collaboration and visibility.
Question 4
What is an API (Application Programming Interface) in supply chain technology?
A.A type of warehouse picking technology
B.A software interface that allows two different systems to communicate and exchange data in real time — enabling integration between ERP, WMS, TMS, and supplier systems without EDI mapping
C.An advanced form of purchase order
D.A planning algorithm for inventory optimization
APIs enable real-time, bidirectional data exchange between systems — modern supply chain platforms use APIs to connect ERP with TMS, WMS, supplier systems, and customer systems, enabling faster integration than traditional EDI.
Question 5
What is 'event management' in supply chain visibility?
A.Planning corporate events and conferences
B.Automatically monitoring supply chain activities, detecting deviations from plan (exceptions/events), alerting the appropriate people, and recommending or triggering corrective actions
C.Managing product launches and marketing events
D.Scheduling production machine maintenance events
Supply chain event management (SCEM) continuously monitors key supply chain milestones (expected vs. actual shipment arrival), detects exceptions, and alerts managers — enabling proactive rather than reactive response.
Question 6
What are the main benefits of electronic supplier connectivity (EDI/API)?
A.Eliminates the need for supplier contracts
B.Reduces manual data entry errors, speeds up order-to-invoice cycle times, enables automatic order processing, and provides real-time status updates from suppliers
C.Replaces the need for supplier performance management

Electronic connectivity (EDI or API) eliminates manual data entry, reduces errors, accelerates cycle times, enables automatic order processing, and provides real-time shipment status — improving both efficiency and accuracy.
Question 7
What is 'track and trace' and why is it important?
A.A marketing tool for customer loyalty programs
B.The ability to track the current location and status of a shipment ('track') and to reconstruct its full movement history ('trace') — critical for customer service, exception management, and regulatory compliance
C.A method for tracing the root cause of quality problems
D.A financial tool for tracking invoice payment status
Track-and-trace provides real-time shipment location (tracking) and historical movement records (tracing) — essential for customer delivery status queries, exception management, and regulatory requirements (e.g., food safety traceability).
Question 8
Which of the following is a key challenge in achieving supply chain visibility?
A.Too much data is available, making it easy to see everything
B.Multiple tiers of suppliers and logistics providers use different, often incompatible systems — creating data silos that must be integrated for end-to-end visibility
C.All supply chain systems are already fully integrated
D.Visibility is only relevant for international supply chains
The primary challenge is data fragmentation — different partners use different systems (ERPs, TMS, WMS) that don't naturally communicate. Achieving end-to-end visibility requires integrating data from many disparate sources.
Question 9
What is 'supply chain transparency' (beyond visibility)?
A.The ability to see internal inventory levels
B.Sharing supply chain information — including supplier practices, environmental impact, and labor conditions — with external stakeholders (customers, investors, regulators) as part of sustainability and CSR commitments
C.Providing real-time data only to the CEO
D.Making all supply chain data public to any party
Transparency goes beyond visibility: it involves voluntarily disclosing supply chain information to external stakeholders — from where products are made, to supplier labor practices and environmental impacts.
Question 10
What is a 'digital supply chain' and how does it differ from a traditional supply chain?
A.A supply chain that only sells digital products
B.A supply chain that leverages digital technologies (IoT, AI, blockchain, cloud) throughout all processes to achieve real-time visibility, predictive analytics, and automated decision-making — vs. paper-based and manual traditional processes
C.A supply chain that operates exclusively online
D.A supply chain managed entirely by robots
A digital supply chain uses technology to create continuous data flows, real-time visibility, and automated decision-making across all processes — dramatically improving speed, accuracy, and responsiveness compared to traditional manual approaches.
Question 11
What does 'data latency' mean and why is it a problem for supply chain visibility?
A.The cost of transmitting data over long distances
B.The time delay between when an event occurs and when data about that event appears in the visibility system — high latency means managers are making decisions based on stale information
C.The amount of data stored in the supply chain database
D.The number of data entry errors in the system
Data latency is the gap between reality and what the system shows. High latency (e.g., inventory updated only weekly) means managers make decisions on outdated information — leading to poor decisions and slow response to exceptions.
Question 12
Which technology enables automatic identification and real-time tracking of goods in a warehouse without line-of-sight scanning?
A.Barcode scanning
B.RFID (Radio Frequency Identification) — reads tags via radio waves at a distance without requiring line-of-sight, enabling bulk scanning of entire pallets or containers
C.EDI electronic documents
D.GPS satellite tracking
RFID's key advantage over barcodes is that it doesn't require line-of-sight — a reader can simultaneously identify multiple items in a container or on a pallet, dramatically speeding up receiving and inventory counts.
